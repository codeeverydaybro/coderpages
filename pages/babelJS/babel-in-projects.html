Setting and Configuring Babel.js in Projects

Babel is a JavaScript Transcompiler, Babel compile newer version of Javascript like ES6 to older version of JS which have wider browser support.

There are many ways by which we can set up Babel locally, but we are going to see how to setup Babel.js in our project using Babel CLI. Checkout setting Babel using other methods here. To begin, here are few prequisites which must be available on your system.

Prequisites

  1. NodeJS: Node.js is a JavaScript runtime built on Chrome's V8 JavaScript engine.
    Download for your computer here.
  2. npm: Node Package Manager, helps install various resuable pieces of code from all over the internet, we will use npm to download BabelJS.
    Checkout this page for more information.

Steps

Following steps will guide us installing and configuring Babel in our project.
If stuck, checkout this page for Babel-CLI installation.

  1. Initialise Project with npm
    To begin, open your terminal and move to the root directory of the project. Now initialise the project using npm.

    $> npm init

    Fill up the necessary information, if needed.
  2. Download Babel using npm
    Now, we will be downloading Babel in our Project folder.
    Move to the root directory of the project in terminal and type in following command:

    $> npm install --save-dev babel-cli

    It may take sometime to complete.
  3. Download Babel presets
    Depending on your project's need, there are many presets that are used with Babel, for example here we are going to transpile ES2015 to old JS, hence we will be using es2015 preset. There are other presets available for babel as well, check them here.
    Go to the root of your project and type:

    $> npm install --save-dev babel-preset-es2015

    NOTE: After these steps your package.json file(in your root directory) must look like this: 
          {
          ...

          "devDependencies": {
              "babel-cli": "^6.x.1",
              "babel-preset-es2015": "^6.x.1",
          }

      }
  4. Writing Build script in package.json
    After completing these steps, we need to write scripts to run babel from Command line, we can manually run the commands from the terminal, but we will be using npm run utitlity to do that.
    In your package.json file, find "scripts": { and add the following line inside curly braces:

    "build":"babel src -d lib"

    Where, src is our source folder and lib is folder where babel will store compiled code.
    Now your package.json file must look like this: 
        {

      ...

      "scripts": {
        "test": "echo \"Error: no test specified\" && exit 1",
        "build": "babel src -d lib"
      },
      "author": "",
      "license": "ISC",
      "devDependencies": {
            "babel-cli": "^6.x.1",
            "babel-preset-es2015": "^6.x.1",
      }

      ...

    }
    NOTE: Always keep source files in folder mentioned in build script. in our build script. In this case it is src folder. Babel will auto generate lib folder with all files containing compiled code.
  5. Apply presets in .babelrc
    Babel is still not going function as we want. We still have to apply presets in order for Babel to work. In your root directory create a file with the name .babelrc.
    Inside the file, add the following line:

    {"preset":["es2015"]}

    If you have more than one presets in your project, just add them in the array.

    {"preset":["es2015","env",...]}

Testing and Running Babel

To test and run Babel, put your file having es2015 code in src folder. In your terminal, run the npm run command to execute babel over the files present in src folder.

$> npm run build

This will compile your code and store it in a folder called lib.
